# 카드 혜택매니저 v7.4.6 Full Subscription Burden


## v7.4.6 Full Subscription Burden
- 카드별 현재 실적기간의 전체 구독 부담액을 반영
- 현재 사용금액에 이미 포함된 구독은 중복 계산하지 않도록 보정
- 전체 구독 부담액 / 현재 포함 추정 / 향후 결제액 표시
- 추가 사용 필요액을 전체 구독 예약 기준으로 계산

기준 베이스: **v7.4.4 Subscription Calendar & UI Refine**

## 핵심 변경
- 카드사 앱에서 확인한 **현재 사용금액은 그대로 유지**합니다.
- 실적 입력 시 **금액 확인 기준일**을 함께 저장합니다.
- 기준일 이후, 현재 카드 실적기간 안에 결제될 구독만 **예정 구독**으로 계산합니다.
- 카드별로 **현재 사용금액 / 남은 예정 구독 / 예상 실적 / 추가 사용 필요액**을 분리 표시합니다.
- 기존의 구독 금액 직접 가산 기능은 제거하고 **현재 사용금액 포함 확인** 방식으로 변경했습니다.

## 계산 기준
- 현대카드: 현재 24일~다음 달 23일 실적기간
- 기타 카드: 현재 월 1일~말일
- 활성 상태이며 연결 카드가 있고 `예정 구독으로 포함`된 구독만 계산
- 결제 예정일이 카드별 현재 사용금액 기준일보다 늦은 항목만 계산
- 이미 `현재 사용금액 포함`으로 확인한 구독은 예정액에서 제외

## 데이터 정책
- 카드 데이터 저장소와 구독 데이터 저장소는 계속 분리합니다.
- 기존 카드 데이터는 유지하며 카드별 `spentAsOfDate`만 추가합니다.
- 자동으로 카드 사용금액을 증가시키지 않습니다.

## 프로젝트 범위
신용카드 사용금액, 실적, 혜택, 구독 결제, 결제 일정, 히스토리, 통계 및 카드 최적화만 다룹니다.
